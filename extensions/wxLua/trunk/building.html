Building wxLua

This article describes the steps required to build wxLua under Linux and Windows. Linux Prerequisites Install the required packages
  • Programs
    • Premake
      • Download this from the main website http://premake.sf.net
      • Install it into your system.
        • $ sudo cp premake /usr/bin
    • CVS
      • Use apt-get to install the cvs package.
    • SVN
      • Use apt-get to install the subversion package.
  • Libraries
    • liblua5.1-0-dev
    • mesa-common-dev
    • libglu1-mesa-dev
Steps (Works for all platforms) Get the wxLua source code
  • Open a terminal and navigate to where you want to store the source for wxLua.
  • Enter the following lines in the terminal to checkout wxLua source.
    • $ cvs -d:pserver:anonymous@wxlua.cvs.sourceforge.net:/cvsroot/wxlua login
    • $ cvs -z3 -d:pserver:anonymous@wxlua.cvs.sourceforge.net:/cvsroot/wxlua co -P wxLua
Generate the build files
  • In the open terminal and navigate to the build directory.
  • Checkout the Premake build files from http://rjpcomputing.homeip.net/svn/lua/libraries/wxlua/build/premake
    • $ svn co http://rjpcomputing.homeip.net/svn/lua/libraries/wxlua/build/premake premake
  • Navigate into the premake directory that was just created
    • $ cd premake
  • Enter one of the following commands to generate the specific version of build files you require.
    • Code::Blocks
      • $ premake --target cb-gcc
    • GNU makefiles
      • $ premake --target gnu
    • Visual C++ 2003 (.NET)
      • $ premake --target vs2003
    • Visual C++ 2005
      • $ premake --target vs2005
    • Visual C++ 2008
      • $ premake --target vs2008
Build wxLua
  • In the open terminal type make.
    • $ make
  • After the build process completes a new wx.so/dll will be created in the bin directory under the root of the wxLua source tree. Copy it where every you need wxLua as a Lua module.
